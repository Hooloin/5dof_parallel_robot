ZG_EXGCODE --扩展G代码

类型

扩展G代码

描述

扩展G代码指令，绑定basic文件定义的GSUB/SUB函数，当执行扩展的G代码时，会调用绑定的GSUB/SUB函数

注：扩展的G代码将会使用GSUB/SUB的语法，详看ZBasic编程手册；

GSUB指令只能单独一行使用，而SUB指令没限制，扩展追加回调函数时，建议使用SUB定义（但无法读取指令参数）。

语法

ZG_ EXGCODE(type, code [,bifappend])

        type：指令类型，"G"-G指令，"M"-M指令，其他-未使用

        code：指令代码

bifappend：是否追加方式，缺省0-不追加，将覆盖原指令处理

1-追加方式，处理完原指令，，以回调方式追加处理GSUB

 

注册扩展指令之后，需要在basic（zar）上添加对应的GSUB，定义名称规则如下：

扩展指令

对应gsub

M指令

直接名称对应，但注意gsub需跟两位以上数字，如：

M6、M06对应的gsub为M06

M100 对应的gsub为M100

G指令（不带小数）

与M指令相同

G5、G05 对应gsub G05

G105 对应gsub G105

G指令（带小数）

整数部分以上相同，小数部分用下划线隔开，如：

G5.1 对应gsub G05_1

适用控制器

通用

例子

ZG_EXGCODE("G", 5)

ZG_EXGCODE("G", 5.1)

END

 

’basic

GLOBAL GSUB G05(X, Y, Z)

・・・

END SUB

 

’SUB定义扩展指令，参数无意义

GLOBAL SUB G05_1()

・・・

END SUB

注册后，在NC文件中直接调用

 

G5 X1 Y1 Z1

G05 X1 Y1 Z1

G5.1 X1 Y1 Z1

G05.1 X1 Y1 Z1
